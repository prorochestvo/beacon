# Task Breakdown — Identity Generation Domain & Dead Code Purge

## Overview

Two intertwined cleanups bundled into a single plan because they share the
same files and verification step:

1. **Promote ID generation to the domain layer.** Today each repository file
   carries a private `generate<Entity>ID` helper that emits identifiers in the
   shared format `<prefix>YYYYMMDDhhmmssZ<nanos>T<UUIDv4-hex>`. The format is
   identical across five entities; only the prefix differs. Centralise it in a
   new package `internal/domain/identity` whose public API exposes a closed
   `Kind` enum, so a typo such as `identity.New("R5")` cannot compile.
2. **Remove the dead symbols** documented in the unused-code audit dated
   2026-06-06. Both production-dead symbols (no caller anywhere) and
   production-dead-but-test-only symbols (covered only by their own tests,
   never reached from any binary) are in scope.

The combined cleanup also removes a stale `build/rulegen` binary and the
accumulated `tmp/after_*.txt` refactor snapshots.

## Assumptions

- The ID format `<prefix>YYYYMMDDhhmmssZ<nanos>T<UUIDv4-hex>` is the current
  contract and must be preserved byte-for-byte. No DB migration is required;
  existing rows keep their existing IDs.
- The repository pattern remains unchanged: callers may pass a record with an
  empty `.ID`, and the repository fills it in inside `Retain*` before insert.
  We are only relocating the helper that mints the string, not changing when
  it is called.
- `generateRateSourceID` is dead in production and stays dead after the move.
  No production caller exists for new `RateSource` minting today, so even
  after `identity.New(identity.KindRateSource)` becomes available, no code
  invokes it. Promotion makes the capability available; wiring it to a caller
  is a separate concern out of scope here.
- `HttpCodeError`, `ColorForPair`, `FiatPalette`, `MetalPalette` are covered
  only by their own tests with no production reader. Their tests are removed
  alongside the production definitions; nothing else in the test files
  depends on the deleted symbols (verified via grep).
- `internal/errors_test.go` retains tests for the other error types
  (`PublicError`, `TraceError`, `StackTraceError`). Only the
  `HttpCodeError`-specific block is removed.

## Tasks

### Task 1: Add `internal/domain/identity` package

- **Description:** Create `internal/domain/identity/identity.go` defining:
  - `type Kind string` — closed enum.
  - Five const Kinds:
    - `KindRateSource           Kind = "RS"`
    - `KindRateValue            Kind = "RV"`
    - `KindRateUserEvent        Kind = "RUE"`
    - `KindRateUserSubscription Kind = "RUS"`
    - `KindExecutionHistory     Kind = "H"`
  - `func New(Kind) string` returning the format described above.
  - Package godoc starts with `// Package identity ...` per CLAUDE.md.
  - Every exported identifier carries a godoc comment that starts with the
    identifier name and ends with a period.
- Create `internal/domain/identity/identity_test.go` with one top-level
  `TestNew` containing subtests per `One Test* per method` rule:
  - `"prefix matches kind"` — for every Kind, `New(k)` starts with the
    Kind's string value.
  - `"output is unique across calls"` — 100 consecutive calls produce 100
    distinct strings.
  - `"format matches contract"` — output matches the regex
    `^<prefix>\d{14}Z\d+T[0-9A-F]+$`.
- **Acceptance criteria:**
  - `make test` green.
  - All five Kind constants compile with the exact two-letter / three-letter
    prefixes used today.
- **Pitfalls:** `time.Now().UTC()` must be the time source — using `time.Now()`
  alone would shift IDs by the local timezone offset and silently change the
  contract. The UUID library is `github.com/twinj/uuid`; its `NewV4().Bytes()`
  returns 16 bytes formatted with `%X` in the existing helpers.
- **Complexity:** Easy.

  with `identity.New(identity.Kind*)`, then delete the private helpers
  themselves. Affected files:

    `:339` — helper.
  - `internal/repository/rateuserevent.go:301` — call site;
    `:507` — helper.
  - `internal/repository/rateusersubscription.go:103` — call site;
    `:407` — helper.
  - `internal/repository/ratevalue.go:189` — call site;
    `:509` — helper.
- Each repository file's imports must add the new package path and may drop
  `time` / `uuid` if no other reference remains (verify per file).
- **Acceptance criteria:**
  - The four `generate*ID` functions no longer exist in `internal/repository/`.
  - All four `Retain*` methods produce the same ID format as before.
  - `make test` green; the repository tests still pass.
- **Pitfalls:** Several repository files import both `time` and `uuid` for
  unrelated reasons (timestamps, ID fields). Do not blindly remove imports —
  re-run `goimports` / verify by build.
- **Complexity:** Easy.

## Execution Order

1 → 2 → 3 → 4. Tasks 2 and 3 could in principle run in parallel, but doing
them serially keeps the diff readable and the verification step trivial.

## Risks

- A reader of `FiatPalette`/`MetalPalette` outside the test file could be
  added between audit and execution. Mitigated by re-grepping before deletion.
- `errors_test.go` benchmark removal could leave the file with no benchmarks
  for one type while keeping them for others — acceptable, no policy violated.
- The new package introduces a circular-import risk only if a repository
  package happens to import another domain package that itself imports
  `identity`. None do today; `identity` has no internal dependencies.

## Trade-offs

- We promote the ID generator but leave entity construction at the repository
  layer (today's pattern: caller passes empty `.ID`, repo fills it). A pure
  DDD move where each domain factory mints its own ID at construction time is
  a larger refactor and is **deliberately out of scope** here.
- `generateRateSourceID` is killed rather than promoted-and-wired. Promoting
  it to `identity.New(identity.KindRateSource)` without a caller would be
  cargo-cult cleanup; the new package exposes the capability, and a future
  task can wire it when a real caller appears.
- The `Kind` enum uses string values that double as the on-disk prefix. This
  conflates two concerns (type discrimination and serialised representation)
  but matches the existing prefix scheme exactly and lets the implementation
  stay a single `fmt.Sprintf`.
